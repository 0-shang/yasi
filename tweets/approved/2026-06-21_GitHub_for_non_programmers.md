GitHub 不只是一个代码网站

如果你是 AI 工具的重度用户但不会写代码，GitHub 可能是你正在错过的最重要的信息入口

很多新的 AI 工具、Agent 项目、MCP 插件、Prompt 模板，刚出来时根本不在 Product Hunt 上，而是在 GitHub 上

对非程序员来说，用好 GitHub 只需要掌握 5 个东西：

---

1. Trending — GitHub 版热门榜

这里展示近期增长最快的项目

想了解最近有哪些 AI 工具正在变热，从这里开始看

Star 数 = 收藏数，越高说明关注度越高，但 README 的质量比 Star 数更能判断一个项目好不好用

---

2. Topics — 按主题找项目

这是一个标签系统。想找某类项目，直接搜关键词：
- agents
- MCP
- LLM
- automation
- prompt-engineering

比如你关心 AI 自动化，订阅 automation 和 workflow 两个标签，就能持续刷到相关项目

---

3. Search — 搜得越具体，结果越有用

不要只搜 "AI"

试试这些：
- AI writing assistant
- MCP server
- local LLM UI
- no code AI agent
- RAG chatbot

把你的真实需求翻译成英文关键词，结果质量会高很多

---

4. README — 项目说明书，决定值不值得用

点进项目后，往下翻看 README

重点看这 3 个东西：
- 项目是做什么的（解决了什么问题）
- 怎么安装使用（有没有截图或 demo）
- 最近有没有更新（长期不更新可能踩坑）

README 写得清楚的项目，通常更靠谱

---

5. Star — 看到好项目先收藏

GitHub 的 Star 就是收藏

对有价值的项目点一下 Star，之后可以从头像菜单里的 Your stars 找回来

就像给自己的工具库做书签，养成习惯

---

一个简单的日常使用路径：

Explore → 看 Trending 发现热门 → Search 搜具体需求 → Topics 扩展相关方向 → 点进项目读 README → 有价值的点 Star

用熟了之后，GitHub 就成了一个持续更新的工具情报站

AI 时代，信息差的一半来源不是你不知道，而是你不知道去哪里看
